## getMany

Retrieves all parent categories with their subcategories.

### Method

```typescript
categories.getMany.query()
```

### Parameters

No input parameters required.

### Response

  Array of parent category objects with nested subcategories
  
  <Expandable title="properties">

### Code Example

```typescript
const categories = await trpc.categories.getMany.query();

categories.forEach(category => {
  console.log(`Parent: ${category.name}`);
  
  category.subcategories.forEach(sub => {
    console.log(`  - ${sub.name}`);
  });
});

// Example output:
// Parent: Electronics
//   - Computers
//   - Phones
//   - Accessories
// Parent: Clothing
//   - Men's
//   - Women's
//   - Kids
```

### Implementation

```typescript
getMany: baseProcedure.query(async ({ ctx }) => {
  const data = await ctx.db.find({
    collection: "categories",
    depth: 1, // Populate Sub-Categories, subcategories.[0] will be a type of "Category"
    pagination: false,
    where: {
      parent: {
        exists: false,
      },
    },
    sort: "name",
  });

  const formattedData = data.docs.map((doc) => ({
    ...doc,
    subcategories: (doc.subcategories?.docs ?? []).map((doc) => ({
      // Because of 'depth: 1' we are confident doc will be a type of "Category"
      ...(doc as Category),
    })),
  }));

  return formattedData;
})
```

### Notes

- This endpoint only returns **parent categories** (where `parent` does not exist)
- Subcategories are automatically populated with `depth: 1`
- Results are sorted alphabetically by name
- Pagination is disabled as this returns the complete category tree
- Useful for building navigation menus and category filters
- Categories have a hierarchical structure with parent-child relationships
