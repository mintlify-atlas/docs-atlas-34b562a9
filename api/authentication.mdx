## Overview

The marketplace uses session-based authentication powered by HTTP-only cookies. Authentication is handled through the `auth` router, which provides procedures for registration, login, and session management.

## Authentication Flow

1. User submits credentials (email/password)
2. Server validates credentials against Payload CMS user collection
3. On success, Payload generates a JWT token
4. Token is stored in an HTTP-only cookie
5. All subsequent requests automatically include the cookie

## Auth Router

The auth router is defined in `src/modules/auth/server/procedures.ts`:

```typescript
import { TRPCError } from "@trpc/server";
import { baseProcedure, createTRPCRouter } from "@/trpc/init";
import { headers as getHeaders } from "next/headers";
import { loginSchema, registerSchema } from "../schemas";
import { generateAuthCookie } from "../utils";

export const authRouter = createTRPCRouter({
  session: baseProcedure.query(async ({ ctx }) => {
    const headers = await getHeaders();
    const session = await ctx.db.auth({ headers });
    return session;
  }),

  login: baseProcedure
    .input(loginSchema)
    .mutation(async ({ input, ctx }) => {
      const data = await ctx.db.login({
        collection: "users",
        data: {
          email: input.email,
          password: input.password,
        },
      });

      if (!data.token) {
        throw new TRPCError({
          code: "UNAUTHORIZED",
          message: "Failed to login",
        });
      }

      await generateAuthCookie({
        prefix: ctx.db.config.cookiePrefix,
        value: data.token,
      });

      return data;
    }),

  register: baseProcedure
    .input(registerSchema)
    .mutation(async ({ input, ctx }) => {
      // Check if username exists
      const existingData = await ctx.db.find({
        collection: "users",
        limit: 1,
        where: {
          username: {
            equals: input.username,
          },
        },
      });

      if (existingData.docs[0]) {
        throw new TRPCError({
          code: "BAD_REQUEST",
          message: "Username already taken",
        });
      }

      // Create tenant for the new user
      const tenant = await ctx.db.create({
        collection: "tenants",
        data: {
          name: input.username,
          slug: input.username,
        },
      });

      // Create user
      await ctx.db.create({
        collection: "users",
        data: {
          email: input.email,
          username: input.username,
          password: input.password, // Automatically hashed by Payload
          tenants: [{ tenant: tenant.id }],
        },
      });

      // Auto-login after registration
      const data = await ctx.db.login({
        collection: "users",
        data: {
          email: input.email,
          password: input.password,
        },
      });

      if (!data.token) {
        throw new TRPCError({
          code: "UNAUTHORIZED",
          message: "Failed to login",
        });
      }

      await generateAuthCookie({
        prefix: ctx.db.config.cookiePrefix,
        value: data.token,
      });
    }),
});
```

## Input Validation

All authentication endpoints use Zod schemas for input validation:

```typescript
import z from "zod";

export const loginSchema = z.object({
  email: z.email(),
  password: z.string(),
});

export const registerSchema = z.object({
  email: z.email(),
  password: z
    .string()
    .min(8, { message: "Password must be at least 8 characters long" })
    .max(32, { message: "Password must not exceed 32 characters" })
    .regex(/[a-z]/, {
      message: "Password must contain at least one lowercase letter",
    })
    .regex(/[A-Z]/, {
      message: "Password must contain at least one uppercase letter",
    })
    .regex(/\d/, { message: "Password must contain at least one number" })
    .regex(/[^a-zA-Z0-9]/, {
      message: "Password must contain at least one special character",
    }),
  username: z
    .string()
    .min(3, "Username must be at least 3 characters")
    .max(63, "Username must be less than 63 characters")
    .regex(
      /^[a-z0-9][a-z0-9-]*[a-z0-9]$/,
      "Username can only contain lowercase letters, numbers and hyphens. It must start and end with a letter or number"
    )
    .refine(
      (val) => !val.includes("--"),
      "Username cannot contain consecutive hyphens"
    )
    .transform((val) => val.toLowerCase()),
});
```

## Cookie Management

Authentication tokens are stored in HTTP-only cookies for security:

```typescript
// src/modules/auth/utils.ts
import { cookies as getCookies } from "next/headers";

interface Props {
  prefix: string;
  value: string;
}

export const generateAuthCookie = async ({ prefix, value }: Props) => {
  const cookies = await getCookies();
  cookies.set({
    name: `${prefix}-token`,
    value: value,
    httpOnly: true, // Prevents JavaScript access
    path: "/",
    // sameSite: "none", // For cross-domain support
    // secure: true, // HTTPS only in production
  });
};
```

### Cookie Security Features

- **httpOnly** - Prevents XSS attacks by blocking JavaScript access
- **path: "/"** - Cookie is sent with all requests
- **Automatic inclusion** - Browser automatically includes cookie in requests

## Client-Side Usage

### Registration

```tsx
"use client";
import { useTRPC } from "@/trpc/client";
import { useState } from "react";

export function RegisterForm() {
  const register = useTRPC.auth.register.useMutation();

  const handleSubmit = async (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const formData = new FormData(e.currentTarget);

    try {
      await register.mutateAsync({
        email: formData.get("email") as string,
        password: formData.get("password") as string,
        username: formData.get("username") as string,
      });
      // User is now registered and logged in
      // Cookie is automatically set
      window.location.href = "/dashboard";
    } catch (error) {
      console.error("Registration failed:", error);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor="username">Username</label>
        <input
          type="text"
          id="username"
          name="username"
          required
          minLength={3}
          maxLength={63}
        />
      </div>
      <div>
        <label htmlFor="email">Email</label>
        <input type="email" id="email" name="email" required />
      </div>
      <div>
        <label htmlFor="password">Password</label>
        <input
          type="password"
          id="password"
          name="password"
          required
          minLength={8}
        />
      </div>
      <button type="submit" disabled={register.isPending}>
        {register.isPending ? "Creating account..." : "Sign up"}
      </button>
      {register.error && (
        <p className="error">{register.error.message}</p>
      )}
    </form>
  );
}
```

### Session Check

```tsx
"use client";
import { useTRPC } from "@/trpc/client";
import { useEffect } from "react";
import { useRouter } from "next/navigation";

export function ProtectedPage() {
  const router = useRouter();
  const { data: session, isLoading } = useTRPC.auth.session.useQuery();

  useEffect(() => {
    if (!isLoading && !session?.user) {
      router.push("/login");
    }
  }, [session, isLoading, router]);

  if (isLoading) {
    return <div>Loading...</div>;
  }

  if (!session?.user) {
    return null;
  }

  return (
    <div>
      <h1>Welcome, {session.user.username}!</h1>
      <p>Email: {session.user.email}</p>
    </div>
  );
}
```

### Server-Side Session Check

```tsx
import { trpc } from "@/trpc/server";
import { redirect } from "next/navigation";

export default async function DashboardPage() {
  const session = await trpc.auth.session();

  if (!session?.user) {
    redirect("/login");
  }

  return (
    <div>
      <h1>Dashboard</h1>
      <p>Welcome back, {session.user.username}!</p>
    </div>
  );
}
```

## Error Handling

The authentication system uses typed tRPC errors:

```typescript
// Possible error codes:
// - UNAUTHORIZED: Invalid credentials or session expired
// - BAD_REQUEST: Validation errors or username already taken

const login = useTRPC.auth.login.useMutation({
  onError: (error) => {
    if (error.data?.code === "UNAUTHORIZED") {
      alert("Invalid email or password");
    } else if (error.data?.code === "BAD_REQUEST") {
      alert(error.message);
    } else {
      alert("An unexpected error occurred");
    }
  },
  onSuccess: () => {
    window.location.href = "/dashboard";
  },
});
```

## Type Safety

All authentication procedures are fully typed:

```typescript
// TypeScript knows the exact shape of inputs and outputs
const login = useTRPC.auth.login.useMutation();

// Input type is inferred from loginSchema
login.mutateAsync({
  email: "user@example.com", // Type: string (email)
  password: "securePassword123!", // Type: string
});

// Return type is inferred from the mutation
const data = await login.mutateAsync({ ... });
// data.user is typed as User from Payload
// data.token is typed as string
```

## Multi-Tenant Support

Registration automatically creates a tenant for each user:

```typescript
// Each new user gets their own tenant
const tenant = await ctx.db.create({
  collection: "tenants",
  data: {
    name: input.username,
    slug: input.username,
  },
});

// User is associated with the tenant
await ctx.db.create({
  collection: "users",
  data: {
    email: input.email,
    username: input.username,
    password: input.password,
    tenants: [{ tenant: tenant.id }],
  },
});
```

## Best Practices

1. **Always validate on server** - Never trust client-side validation alone
2. **Use HTTP-only cookies** - Prevents XSS attacks
3. **Check session server-side** - For protected pages, verify session in Server Components
4. **Handle errors gracefully** - Provide clear feedback to users
5. **Redirect after auth** - Use `window.location.href` or `router.push()` after successful auth
6. **Password requirements** - Enforce strong passwords with validation schema
7. **Rate limiting** - Consider adding rate limiting to prevent brute force attacks (not shown in code)

## Session Persistence

Sessions are automatically persisted across requests:

- Cookie is sent with every request to `/api/trpc`
- Payload validates the token on each request
- Session data is available in the context via `ctx.db.auth({ headers })`
- No manual token management required on the client
