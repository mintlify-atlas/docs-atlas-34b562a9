## Get Started in 10 Minutes

This quickstart will take you from zero to a functioning multi-vendor marketplace with your first vendor storefront and product. Perfect for evaluating the platform or rapid prototyping.

<Note>
  This guide uses simplified setup for quick evaluation. For production deployment, see the [Installation Guide](/installation).
</Note>

## Step 1: Clone and Install

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/yourusername/multi-vendor-marketplace.git
    cd multi-vendor-marketplace
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
    
    This installs all required packages including:
    - Next.js 15 and React 19
    - Payload CMS and multi-tenant plugin
    - tRPC and React Query
    - shadcn/ui components
    - Stripe SDK (payments handled in production)
  </Step>

  <Step title="Set Up Environment Variables">
    Create a `.env` file in the root directory:
    
    ```bash .env
    # Database
    DATABASE_URI=mongodb://localhost:27017/marketplace
    
    # Payload CMS
    PAYLOAD_SECRET=your-random-secret-key-here
    
    # Next.js
    NEXT_PUBLIC_SERVER_URL=http://localhost:3000
    
    # Stripe (optional for quickstart)
    # STRIPE_SECRET_KEY=sk_test_...
    # NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...
    ```
    
    <Warning>
      Generate a secure `PAYLOAD_SECRET` using: `openssl rand -base64 32`
    </Warning>
  </Step>
</Steps>

## Step 2: Initialize the Database

<Steps>
  <Step title="Run Database Migrations">
    ```bash
    npm run db:fresh
    ```
    
    This creates the necessary collections in MongoDB:
    - `users` - User accounts and authentication
    - `tenants` - Vendor storefronts
    - `products` - Product catalog
    - `categories` - Product organization
    - `tags` - Product tagging
    - `media` - Image and file uploads
  </Step>

  <Step title="Seed Sample Data (Optional)">
    ```bash
    npm run db:seed
    ```
    
    This populates your database with:
    - Sample categories (Electronics, Fashion, Digital Products)
    - A demo vendor account
    - Sample products
    - Test tags
  </Step>
</Steps>

## Step 3: Start the Development Server

```bash
npm run dev
```

Your marketplace is now running at:
- **Main Platform**: [http://localhost:3000](http://localhost:3000)
- **Admin Dashboard**: [http://localhost:3000/admin](http://localhost:3000/admin)
- **API Endpoint**: [http://localhost:3000/api/trpc](http://localhost:3000/api/trpc)
- **GraphQL Playground**: [http://localhost:3000/api/graphql-playground](http://localhost:3000/api/graphql-playground)

## Step 4: Create Your First Vendor Account

<Steps>
  <Step title="Navigate to Sign Up">
    Visit [http://localhost:3000/sign-up](http://localhost:3000/sign-up)
  </Step>

  <Step title="Register a New Vendor">
    Fill in the registration form:
    
    <CodeGroup>
    ```typescript Form Fields
    {
      username: "mystore",        // Your subdomain slug
      email: "vendor@example.com",
      password: "SecurePass123!"   // Min 8 characters
    }
    ```
    </CodeGroup>
    
    <Note>
      The `username` field becomes your storefront subdomain. For example, `mystore` creates `mystore.market.com` (in production).
    </Note>
  </Step>

  <Step title="Automatic Tenant Creation">
    Upon registration, the platform automatically:
    
    1. Creates your user account
    2. Generates a tenant record with your username as the slug
    3. Creates a Stripe Connect account ID (placeholder in dev mode)
    4. Associates your user with the tenant
    
    Here's what happens behind the scenes:
    
    ```typescript src/modules/auth/server/procedures.ts
    export const authRouter = createTRPCRouter({
      register: baseProcedure
        .input(registerSchema)
        .mutation(async ({ ctx, input }) => {
          // Create user account
          const user = await ctx.db.create({
            collection: "users",
            data: {
              email: input.email,
              password: input.password,
              username: input.username,
              roles: ["user"],
            },
          });
          
          // Create tenant/storefront
          const tenant = await ctx.db.create({
            collection: "tenants",
            data: {
              name: `${input.username}'s Store`,
              slug: input.username,
              stripeAccountId: `acct_${generateId()}`, // Real Stripe ID in prod
              stripeDetailsSubmitted: false,
            },
          });
          
          // Link user to tenant
          await ctx.db.update({
            collection: "users",
            id: user.id,
            data: {
              tenants: [{ tenant: tenant.id }],
            },
          });
          
          return { user, tenant };
        }),
    });
    ```
  </Step>
</Steps>

    ```typescript src/modules/checkout/store/use-cart-store.ts
    import { create } from "zustand";
    import { persist } from "zustand/middleware";
    
    export const useCartStore = create(
      persist(
        (set) => ({
          items: [],
          addItem: (product) =>
            set((state) => ({
              items: [...state.items, product],
            })),
          removeItem: (productId) =>
            set((state) => ({
              items: state.items.filter((item) => item.id !== productId),
            })),
        }),
        { name: "cart-storage" }
      )
    );
    ```

## Understanding the Architecture

Now that you have a running marketplace, let's understand how it works:

<AccordionGroup>
  <Accordion icon="database" title="Multi-Tenant Data Isolation">
    Products are automatically scoped to tenants using Payload's multi-tenant plugin:
    
    ```typescript src/payload.config.ts
    multiTenantPlugin<Config>({
      collections: {
        products: {},  // Products inherit tenant scoping
      },
      userHasAccessToAllTenants: (user) =>
        Boolean(user?.roles?.includes("super-admin")),
    })
    ```
    
    When a vendor creates a product, it's automatically associated with their tenant. Queries are automatically filtered:
    
    ```typescript
    // Automatically filtered to current user's tenant
    const products = await payload.find({
      collection: "products",
    });
    ```
  </Accordion>

  <Accordion icon="route" title="Subdomain Routing">
    In production, subdomains are handled by middleware:
    
    ```typescript middleware.ts (production)
    export function middleware(request: NextRequest) {
      const hostname = request.headers.get("host");
      const subdomain = hostname?.split(".")[0];
      
      if (subdomain && subdomain !== "www") {
        // Rewrite to tenant route
        return NextResponse.rewrite(
          new URL(`/tenants/${subdomain}${request.nextUrl.pathname}`, request.url)
        );
      }
    }
    ```
    
    In development, access via `/tenants/[slug]` paths.
  </Accordion>

  <Accordion icon="credit-card" title="Stripe Connect Flow">
    The platform uses Stripe Connect for multi-vendor payments:
    
    1. **Onboarding**: Vendors complete Stripe Connect onboarding
    2. **Payment**: Customer pays through platform
    3. **Fee Collection**: Platform fee is automatically deducted
    4. **Payout**: Remaining amount goes to vendor's bank account
    
    ```typescript
    const tenant = await payload.findByID({
      collection: "tenants",
      id: tenantId,
    });
    
    const paymentIntent = await stripe.paymentIntents.create({
      amount: 2999,
      currency: "inr",
      application_fee_amount: 299, // 10% platform fee
      transfer_data: {
        destination: tenant.stripeAccountId,
      },
    });
    ```
  </Accordion>

  <Accordion icon="arrows-split-up-and-left" title="Type-Safe API with tRPC">
    All API calls are type-safe from client to server:
    
    ```typescript Client Usage
    "use client";
    import { useTRPC } from "@/trpc/client";
    
    export function ProductList() {
      const trpc = useTRPC();
      
      const { data: products } = trpc.products.getMany.useQuery({
        limit: 12,
        category: "digital-products",
      });
      
      // products is fully typed!
      return products?.docs.map((product) => (
        <ProductCard key={product.id} product={product} />
      ));
    }
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-exclamation" title="MongoDB Connection Failed">
    **Error**: `MongooseServerSelectionError: connect ECONNREFUSED`
    
    **Solutions**:
    - Ensure MongoDB is running: `mongod` or check Docker container
    - Verify `DATABASE_URI` in `.env` matches your MongoDB connection string
    - For MongoDB Atlas, check IP whitelist and credentials
  </Accordion>

  <Accordion icon="circle-exclamation" title="Port Already in Use">
    **Error**: `Error: listen EADDRINUSE: address already in use :::3000`
    
    **Solutions**:
    - Kill the process using port 3000: `lsof -ti:3000 | xargs kill`
    - Or use a different port: `PORT=3001 npm run dev`
  </Accordion>

  <Accordion icon="circle-exclamation" title="Type Generation Errors">
    **Error**: TypeScript errors about missing `payload-types.ts`
    
    **Solution**:
    ```bash
    npm run generate:types
    ```
    This generates TypeScript types from your Payload collections.
  </Accordion>

  <Accordion icon="circle-exclamation" title="Admin Login Issues">
    **Error**: Cannot access `/admin` or login fails
    
    **Solutions**:
    - Ensure you registered via `/sign-up` first
    - Check that `PAYLOAD_SECRET` is set in `.env`
    - Clear browser cookies and try again
  </Accordion>
</AccordionGroup>
